## Challenge

- Competition: `UVT`
- Challenge: `satellua`
- Category: `Reverse Engineering`


## Solve Path

1) Category detection
- `rev` (reverse engineering), high confidence.
- Evidence: stripped ELF64 binary, embedded VM bytecode/data, and flag assembly logic in disassembly.

2) Attack strategy
- Minimal hypothesis: flag chars are generated by periodic sampling of an internal 64-bit sequence, then XORed with a static table.
- Validation: recover sequence rule from runtime values, verify against many observed samples, then compute sampled positions directly.

3) Execution steps
- Located flag path in handler around `0x4058be`: `flag[i] = collapsed_byte ^ table[i]`, with period gate `0x111088`.
- Extracted runtime integer values (`rdi`) at handler hits; found exact recurrence:
  - `x1 = 0x1fff000`
  - `x_{n+1} = mix(x_n + 0x9E3779B97F4A7C15)` where `mix` is SplitMix64 finalizer.
- Verified model against 109 observed collapsed bytes (perfect match).
- Computed `collapsed(x_{k*0x111088})` for `k=1..` and XORed with table bytes at `0x4227e0`.
- Decoded message resolves to the full flag string.

4) FLAG
- `UVT{R3turn_8y_Thr0w_Del1v3r3r}`

5) Writeup
- The binary intentionally throttles progress and only reveals characters at large iteration intervals. Reversing the handler showed the XOR-table flag construction and the exact periodic gate. By deriving and validating the integer generator mathematically, I computed the required checkpoint bytes offline and reconstructed the final flag without brute-forcing millions of slow cycles.
